# 의미론적 충돌 탐지 개선

## 배경

문서를 수정해도 의미론적 충돌이 재검사되지 않았고, 비교 화면에 문서 전체 diff가 표시되어 실제 충돌 지점을 찾기 어려웠다. 충돌 설명이 영어로 생성되거나, 구조 설명과 운영 현황처럼 서로 책임 범위가 다른 문서를 충돌로 오인하는 문제도 있었다.

## 변경점

- 문서 수정 시 의미론적 충돌을 다시 검사하고 해소된 참조를 양쪽 문서에서 제거한다.
- 충돌 설명은 항상 한글로 생성한다.
- 동일한 대상·속성·범위·시점에서 동시에 참일 수 없는 명시적 주장만 충돌로 인정한다.
- 구조와 운영 상태, 정책과 구현, 기능 존재와 실제 사용을 구분한다.
- 정보 누락이나 문서에 없는 추론은 반대 주장으로 취급하지 않는다.
- `conflict`, `complementary`, `uncertain`으로 분류하고 검증된 `conflict`만 저장한다.
- 양쪽 문서의 실제 근거 문장이 원문에 존재할 때만 충돌을 확정한다.
- 비교 화면에는 전체 diff 대신 대상·속성·범위·시점과 양쪽 근거 문장을 표시한다.

## 결과

문서의 목적이나 관점이 다를 뿐인 사례는 충돌에서 제외되며, 사용자는 실제로 모순되는 문장과 판단 기준을 바로 확인할 수 있다.
